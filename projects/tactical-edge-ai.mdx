## Overview

The Tactical Edge AI Emulator demonstrates Defense Unicorns' strategy for deploying modern software to air-gapped, disconnected tactical networks. This fully offline deployment validates the precise DoD packaging pipeline required to bypass Authority to Operate (ATO) bottlenecks and physically deliver AI capabilities to contested environments.

<Note>
This project validates deployment strategies for **JADC2 (Joint All-Domain Command and Control)** architectures and tactical edge computing in denied environments.
</Note>

## Deployment Strategy

<Steps>
  <Step title="Package Application">
    Bundle the AI application and all dependencies into a Zarf package that can be physically transported
  </Step>
  <Step title="Physical Transfer">
    Move the packaged application to the air-gapped network via secure physical media
  </Step>
  <Step title="Deploy to K3s">
    Extract and deploy the Zarf package to the lightweight Kubernetes cluster running on tactical hardware
  </Step>
  <Step title="Validate Operation">
    Verify the AI emulator is operational without any external network connectivity
  </Step>
</Steps>

## Authority to Operate (ATO) Bypass

Traditional DoD software deployment faces significant delays due to Authority to Operate requirements. This architecture addresses those challenges:

<Accordion title="Challenge: Traditional ATO Timeline">
  Conventional software deployments can take 6-18 months to receive ATO approval, delaying critical capabilities from reaching warfighters.
</Accordion>

<Accordion title="Solution: Pre-Packaged, Validated Bundles">
  By using Zarf to create pre-validated, immutable packages, the deployment bypasses many ATO bottlenecks. The entire application stack is validated once and deployed multiple times without network connectivity requirements.
</Accordion>

<Accordion title="Impact: Rapid Capability Delivery">
  This approach enables rapid delivery of modern AI/ML capabilities to tactical units operating in contested or denied environments where traditional cloud connectivity is unavailable.
</Accordion>

## Technical Implementation

### Air-Gapped Architecture

```yaml
# Example Zarf package configuration
kind: ZarfPackageConfig
metadata:
  name: tactical-ai-emulator
  version: 1.0.0
  description: Air-gapped AI deployment for tactical edge

components:
  - name: k3s-cluster
    required: true
    manifests:
      - name: k3s-config
        files:
          - cluster-config.yaml
  
  - name: ai-models
    required: true
    images:
      - ai-inference-engine:latest
      - model-server:v2.1
    
  - name: edge-runtime
    required: true
    charts:
      - name: edge-orchestrator
        version: 1.2.3
```

### Kubernetes Edge Configuration

The K3s cluster is configured for minimal resource consumption while maintaining production-grade reliability:

```bash
# K3s installation for tactical edge
curl -sfL https://get.k3s.io | sh -s - \
  --write-kubeconfig-mode 644 \
  --disable traefik \
  --disable servicelb \
  --node-taint edge=true:NoSchedule
```
